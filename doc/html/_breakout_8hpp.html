Detailed Description

The main header file of the game program.

Author
Ziqing Mao
Version
1.0

@email mao.zi(at)northeastern.edu

Date
21-Feb-2021

This header defines different object classes of the breakout game. Instances, even from different classes, can share the same texture that only has one copy in the memory.
In the light of flyweight pattern, instances only contain their unique information like position and movement states, but textures which can be shared by different instances in the same texture demand. This is accomplished by a single resource manager that holds all the resources, with instances of object only contain the pointer. The resource manager will search in a map before allocating new memory to ensure the uniqueness when it is called for loading a certain resource.
The calculation of object movement and collision is supported by TinyMath library, which includes vector addition, scalar multiplication and so on.
